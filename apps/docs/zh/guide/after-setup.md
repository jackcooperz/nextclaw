# 配置后做什么

当你已经完成 Provider + 默认模型配置，建议马上做这 5 件事，把 NextClaw 从“可用”变成“真的在用”。

如果你是“刚安装、还没配 Provider”，先看这篇：

- [安装后第一步：先选接入方式（Qwen Portal 或 API Key）](/zh/guide/tutorials/provider-options)

## 1. 先确认运行健康

```bash
nextclaw status --json
nextclaw doctor --json
```

这一步能快速确认进程、配置与关键依赖都正常。

## 2. 做一个 10 分钟内能看到价值的任务

在 UI 里直接发一个你每天都会做的请求，例如：

- 总结今天待办
- 生成一条工作汇报草稿
- 把一段需求整理成可执行清单

重点不是“复杂”，而是让你在第一天建立稳定使用习惯。

## 3. 接入一个你常用的消息渠道

优先从你每天都打开的平台开始（如 Discord / Telegram / Slack）。

- 渠道总览：[渠道](/zh/guide/channels)
- 对应实操：[教程](/zh/guide/tutorials)

## 4. 配一个最小自动化任务

给自己加一个低风险、高频的定时任务，比如“每天 9:30 提醒当天重点”。

- 文档入口：[Cron 与 Heartbeat](/zh/guide/cron)

## 5. 看资源导航，抄一条现成方案

如果你不知道下一步做什么，优先参考生态项目与案例，再按你的场景改造。

- 资源页：[生态资源](/zh/guide/resources)

## 外部教程与分享（后续补充）

当前先保持克制，不预置 OpenClaw 文档与仓库链接。后续按你的节奏手动补充。

## 推荐顺序（不纠结版）

1. 健康检查
2. 做一个真实小任务
3. 接入一个消息渠道
4. 加一个定时自动化
5. 从资源页复制一条可落地方案
